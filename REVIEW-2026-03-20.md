# MeClaw Review — 2026-03-20

## Numbers

- 33 SQL files, 8,129 lines of code
- 29 tables, 74 functions (46 plpgsql, 22 plpython3u, 6 sql)
- 108 tests (69 smoke + 39 extended), all green
- 29 commits, 1 day of development
- AGE Graph: 22 nodes (7 Entity, 6 Bee, 3 Event, 2 Agent, 2 Channel, 2 Hive), 23 edges
- 27 Brain Events, 7 Entities, 5 Prototypes, 99 Messages, 1 Execution Plan

## Strengths

1. **Architecture vision** — Channel-level extraction vs. agent-level brain, 5-layer storage, AIEOS integration. Clearly differentiates from Mem0/Zep/EverMemOS.
2. **Postgres-only USP holds** — AGE + pgvector + pg_search + pg_cron + pg_background + plpython3u in a single container.
3. **Swarm pipeline functional** — 4-step DAG, $0.0006, end-to-end.
4. **Test coverage decent for vibe-coded** — 108 real assertions, not just skeleton tests.

## Weaknesses

1. **Breadth over depth** — ~45 min per phase. Prototyping, not engineering.
2. **Graph nearly empty** — 7 entities, 5 INVOLVED_IN, 2 RELATES_TO with 27 events. Test data is "test"/"ping".
3. **plpython3u security risk** — 22 functions with full OS access (requests, os, subprocess).
4. **Error handling is optimistic** — EXCEPTION WHEN OTHERS + Warning everywhere. Bad for subtle bugs.
5. **Happy-path tests only** — No tests for: API timeout, corrupt data, concurrent access, trigger loops, invalid IDs.
6. **CTM retrieval unvalidated** — With 27 events, entropy always converges immediately. No meaningful test.
7. **Swarm not in live flow** — concierge_bee/planner_bee only as manual functions, not in trigger chain.
8. **context_bee_v3 not wired up** — v2 still running live.
9. **Hebbian learning empty** — 0 prototype_associations. Too few feedback events.
10. **Cost tracking incomplete** — Only DAG tracked. extract_bee/embedding/sentiment LLM calls not tracked.

## Assessment

Impressive prototype, premature release. Architecture is solid, all pieces exist, but nothing has seen real traffic.
